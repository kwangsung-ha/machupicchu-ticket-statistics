# 마추픽추 티켓 현황 모니터링 시스템 - 단계별 구현 계획 (Implementation Plan)

프로젝트를 검토하기 쉬운 작은 단위(Step)로 쪼개어 계획을 수립했습니다. 각 Step이 완료될 때마다 사용자님의 검토를 받고 다음 단계로 넘어갑니다.

## 단계 (Phases & Steps)

### Step 1: 대상 사이트 크롤링 프로토타입 개발 (가장 중요)
*   **목표**: 페루 문화부 사이트(TuBoleto)에서 실제로 데이터를 가져올 수 있는지 검증.
*   **작업 내용**:
    *   간단한 Python 스크립트 작성.
    *   사이트 구조 분석 (`httpx` + `BeautifulSoup` 또는 동적 렌더링일 경우 `Playwright` 사용).
    *   특정 날짜의 각 코스별 총 티켓, 예매 수, 잔여 수 추출 검증.
*   **검토 포인트**: 데이터를 성공적으로 파싱하여 콘솔에 출력하는지 확인.

### Step 2: 백엔드 뼈대 구성 및 데이터베이스 스키마 설계
*   **목표**: FastAPI 기반 웹 서버와 SQLite 데이터베이스 연결.
*   **작업 내용**:
    *   Python 가상 환경 설정 및 패키지 설치 (`fastapi`, `uvicorn`, `sqlmodel`, `apscheduler` 등).
    *   `sqlmodel`을 이용한 데이터베이스 스키마 작성 (`Circuit`, `AvailabilityLog`).
    *   데이터베이스 생성 및 연동 테스트.
*   **검토 포인트**: 코드가 정상 실행되고 SQLite 데이터베이스 파일(`sqlite.db`)과 테이블이 생성되는지 확인.

### Step 3: 스케줄러 연동 및 데이터 저장 자동화
*   **목표**: Step 1에서 만든 크롤러를 FastAPI 백엔드에 통합하고, 30분 주기로 실행.
*   **작업 내용**:
    *   `APScheduler`를 FastAPI 시작 이벤트에 연동.
    *   30분마다 크롤러를 실행하여 가져온 데이터를 SQLite에 저장하도록 로직 연결.
*   **검토 포인트**: 백엔드 서버를 띄웠을 때 정해진 주기마다 크롤러가 동작하고 데이터베이스에 쌓이는지 확인.

### Step 4: 백엔드 API 구현
*   **목표**: 프론트엔드에서 사용할 REST API 제공.
*   **작업 내용**:
    *   `GET /api/availability/today`: 오늘 기준 최신 티켓 현황.
    *   `GET /api/availability/history`: 특정 코스의 최근 수집 이력 추이(통계용).
*   **검토 포인트**: 브라우저나 API 클라이언트를 통해 데이터가 JSON 형태로 정상 반환되는지 확인.

### Step 5: 프론트엔드 (React 대시보드) 개발
*   **목표**: 사용자에게 시각적으로 데이터를 보여주는 웹 UI 구축.
*   **작업 내용**:
    *   React(Vite 등) 프로젝트 초기화.
    *   FastAPI의 API를 호출하여 화면에 테이블 및 차트(`recharts` 등 활용) 표시.
*   **검토 포인트**: 화면 UI 확인 및 데이터가 제대로 시각화되는지 검토.

### Step 6: 통합 및 단일 Dockerfile 배포 준비
*   **목표**: 프론트엔드와 백엔드를 합치고 Docker 컨테이너 1개로 실행 가능하도록 구성.
*   **작업 내용**:
    *   React 빌드 결과물을 FastAPI가 정적 파일(Static Files)로 서빙하도록 라우팅 추가.
    *   단일 `Dockerfile` 작성 (Node.js 이미지로 빌드 후 Python 이미지로 실행하는 Multi-stage build).
*   **검토 포인트**: `docker build` 및 `docker run` 명령어로 서비스가 한 번에 실행되는지 확인.
